# 0005 — vocab-review 用户对象修正为"家长"

- **日期**：2026-04-26
- **状态**：✅ 生效中
- **影响范围**：vocab-review（IA、文案、交互模型）

## 背景

vocab-review 早期默认假设是"孩子直接打开 App 做题"。底栏 6 个 tab（首页 / 单词本 / 任务单 / 批改 / 录入 / 统计 / 设置）都是基于这个假设排的：孩子点首页看任务、点任务单做题、点统计看自己。

跑了一段时间后用户复盘："整个功能分区还是比较的杂乱，你能重新调用 gstack 给我重新规划一下吗"。

AI 第一版重构提了"今日 / 词库 / 复盘 / 设置"4 tab，仍然按"孩子用"思路排（"今日"是孩子做题入口）。

**用户当面纠正**：「这个主要是给家长用的，不给孩子用。用户对象要搞清楚。」

## 选项

| 方案 | 含义 |
|---|---|
| A. 保持"孩子用" | App 内做题 / 自我激励 / 进度可视化都对孩子 |
| B. 切换到"家长用" | App 是家长备课 + 出题 + 批改 + 复盘工具，孩子在纸上做 |
| C. 双角色 | 同一界面切换家长 / 孩子模式 |

## 决定

选 **B：用户对象 = 家长**。

明确的家长动线：

```
每周 1 次：录入这周课本新词（备课）
每天 1 次：看今日清单 → 出题给孩子 → 批改 → 错词回炉
每周末：看孩子掌握度 / 高频错词 / 决定下周策略
偶尔：调设置、备份数据
```

孩子做题载体：**纸上写**（家长打印题卡 / 口头听写）。
批改方式：**OCR 上传识别错题**。

## 理由

**用户视角**：

- 孩子做题在屏幕上 vs 在纸上，纸上更接近 KET 真实考试场景
- 减少屏幕时间，家长更愿意接受
- 家长真正需要的是"备课工作台"，不是"孩子学习 App"

**工程视角**：

- 用户对象不同，IA 完全不同。一旦定错，所有页面文案、跳转、信息密度都要返工
- 切到"家长"后，"练习"模块（在线刷题）从主线降级为辅助 / 待删
- "批改"从模糊的 tab 变成清晰的"OCR 识别错题"功能

**关键学习**：

> 在做 IA 重构前，**必须先把用户对象定死**。AI 默认会按"用户 = 终端体验者"假设走，对家庭场景这种"代理人使用"模式（家长用 App 服务孩子）会判断错。

## 与 AI 讨论摘要

- AI 第一版重构提的"今日 / 词库 / 复盘 / 设置"4 tab 结构本身没问题，但语义错位（按孩子心智组织内容）
- 用户纠正后 AI 重新组织：今日 = 家长教学清单（不是孩子做题入口）；批改 = OCR 流水线一步（不是独立 tab）
- 用户进一步澄清两个关键点：(1) 孩子在纸上做（非 App 内做题）；(2) 批改 = OCR 上传识别错题。这两点决定了"今日"页的形态

## 状态

后续如果出现"孩子也想直接用 App 做题"的需求，需要新开 ADR 评估"双角色"成本。
