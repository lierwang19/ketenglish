# 0006 — vocab-review IA 重构为家长视角 4 tab

- **日期**：2026-04-26
- **状态**：✅ 生效中
- **影响范围**：vocab-review

## 背景

继 [ADR 0005](0005-用户对象=家长不是孩子.md) 把用户对象明确为"家长"后，要重新组织 IA。

旧 IA：底栏 6 项 = 首页 / 单词本 / 任务单 / 批改 / 录入 / 统计 / 设置（实际有 7 个隐藏页 section），用户反馈"杂乱"。

主要问题：

1. **任务粒度错位** —— 单词本 / 录入 / 任务单 / 批改 实际都围绕"单词数据"，但被拆成 4 个一级入口
2. **命名模糊** —— "批改"看不出是 OCR 还是手判；"单词本"和"录入"边界不清
3. **移动端违反 5 项原则** —— iOS HIG / Material 都建议底栏 ≤5 项
4. **任务流断裂** —— 家长每天的"看今日清单 → 出题 → 批改"被强行拆到 3 个 tab

## 选项

| 方案 | 描述 |
|---|---|
| A. 维持 6 tab，只调文案 | 治标不治本 |
| B. 收敛到 4 tab（今日 / 词库 / 复盘 / 设置），子页面用页面内入口而非底栏 | 符合移动端规范，按家长动线组织 |
| C. 5 tab + 抽屉菜单 | 多一个交互层，对家庭场景过重 |

## 决定

选 **B：4 tab + 隐藏路由 + 父 tab 高亮**。

| Tab | 含义 | 收纳的旧页 |
|---|---|---|
| **今日**（默认）| 今日清单 + 导出题卡 + 上传答卷批改 | 首页 + 任务单 + 批改 |
| **词库** | 周次列表 + 单词管理 + 录入入口 | 单词本 + 录入 |
| **复盘** | 掌握度、高频错词、周报 | 统计 |
| **设置** | 每日量、数据导入导出 | 设置 |

落地：

- `index.html` 底栏 6 项 → 4 项
- 路由 `/tasks` `/grading` `/import` 仍注册可直达，但不在底栏显示
- `app.js` 中 `NAV_ROUTE_MAP` 把隐藏路由映射到父 tab（`tasks` / `grading` → `/`，`import` → `/words`），避免进入子页面后底栏"丢失定位"
- `renderHome` hero actions 加第三个按钮 **"上传答卷批改"** → `/grading`
- 各页面 setHeaderTitle 改新名（首页→今日 / 单词本→词库 / 复习统计→复盘）
- 页面正文里 6 处旧文案残留同步清理（`单词本` → `词库`、`统计加载失败` → `复盘加载失败`）
- `sw.js` cache 版本 `v10` → `v11`

题卡 PDF 导出功能作为独立增量，不在本次提交范围。

## 理由

**用户视角**：

- 家长每天的核心动线"今日清单 → 出题 → 批改"压缩到一个 tab，零跳转
- "词库"覆盖所有"管理单词"场景（看 / 改 / 删 / 录入），心智一致
- 4 项底栏在手机上每个按钮区域更大，误触率更低

**工程视角**：

- 路由层不变，渲染层不变，只动导航和入口配置，改动量小
- 隐藏路由保留意味着旧链接 / 收藏夹仍能用，零迁移负担
- 父 tab 高亮逻辑用 NAV_ROUTE_MAP 一次性配置，新增隐藏路由也是改一行

## 与 AI 讨论摘要

- AI 第一版方案错在"用户对象"未明确（详见 [ADR 0005](0005-用户对象=家长不是孩子.md)）
- 第二版基于"家长 + 纸上做题 + OCR 批改"三点明确后定稿
- AI 提议"批改"独立 tab，被否定 —— 批改是"今日"动线里的一步，拆出去反而割裂
- AI 提议"练习"在线刷题模块直接砍掉，用户接受，但本次提交未删（V1.x 再清理）
- 提交前交 Codex 复核：0 个 P1 / 0 个 P2 / 1 个 P3（旧文案残留），P3 已修
