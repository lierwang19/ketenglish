# KET 单词滚动复习软件需求书

V1.0｜面向 KET 备考家庭的词汇滚动复习与错词管理系统

| **项目定位** | 旧词滚动复习机 + 会写词强化工具                          |
|--------------|----------------------------------------------------------|
| **核心对象** | KET 备考中的小学生家庭                                   |
| **使用方式** | 孩子保留自己的新词记忆方式，系统负责滚动复习与错词回炉   |
| **交付对象** | 可直接提供给前端 / 后端 / 产品开发人员作为 V1.0 实施依据 |

一句话总结：本项目不是做完整英语学习平台，而是做“围绕老师周任务、自动安排旧词复习、持续清理错词”的轻量单词系统。

# 1. 项目背景

- 孩子正在备考 KET，同时在校老师每周会布置约 100 个单词任务，其中约 30
个要求会写，约 70 个要求做到“看到英文能说出中文”。

- 孩子已经形成自己的新词背诵方式，因此产品不建议强行替代新词学习过程，而应重点解决“背过就忘、旧词越积越多、会写词难巩固、家长难安排复习”的问题。

- 本产品的核心定位为：单词滚动复习与错词管理系统，而非全功能英语学习
App。

# 2. 项目目标

- 支持每周录入老师布置的新单词，并区分认读词与会写词。

- 根据规则自动生成每日复习任务，减少家长手工安排成本。

- 对错词进行自动回炉，对会写词进行高频强化。

- 对已掌握单词降低复习频率，避免第二周、第三周后任务量失控。

- 提供家长可视化统计，便于查看掌握情况、错词情况和每周复盘结果。

# 3. 产品核心原则

- 目标上对齐老师要求，执行上采用“每天少量推进 + 每天复习 +
周末复盘”的滚动方式。

- 尊重孩子现有的新词学习方法，系统主要接管旧词复习与回炉。

- 采用分层管理：不会的多见，会的少见；红词高频、黄词中频、绿词低频。

- 认读词与会写词分开管理，采用不同练习形式与晋级规则。

- 每日任务量必须可控，避免系统越用越重。

# 4. 用户角色与使用边界

## 4.1 家长端

- 录入本周单词，支持批量导入与手动编辑。

- 标记单词类型：认读词 / 会写词。

- 标记哪些词已进入复习池。

- 查看每日复习情况、错词情况、每周进度与掌握分层。

## 4.2 孩子端

- 完成系统自动生成的“今日复习任务”。

- 进行认读练习、拼写练习和错词回炉。

- 查看今日完成情况、连续打卡与简单成就反馈。

## 4.3 V1.0 边界

- V1.0 不要求在系统中完成完整新词学习。

- V1.0 不做 AI 讲解、口语评分、复杂游戏剧情、多人社交排名。

- V1.0 重点是把“已学过的词”纳入长期可持续的复习系统。

# 5. 核心业务流程

- 家长按周录入老师布置的新单词。

- 每个单词标记为认读词或会写词。

- 家长在孩子完成新词学习后，点击“进入复习池”。

- 系统从该日期开始为单词安排后续滚动复习。

- 孩子每天打开系统，完成“今日任务”。

- 系统根据答题结果更新该单词的掌握等级与下次复习时间。

- 错误单词自动进入高优先级回炉池，已掌握单词转入低频抽查池。

# 7. 详细功能需求

## 7.1 单词管理模块

- 支持手动新增单词，字段包括：英文单词、中文释义、词性（可选）、例句（可选）、所属周次、所属单元/主题（可选）、单词类型、录入日期、进入复习池日期、备注（可选）。

- 支持 Excel / CSV
批量导入，最少字段为：英文、中文、单词类型、所属周次。

- 支持编辑单词内容、所属周次、备注以及单词类型。

- 每个单词需要有状态字段：未进入复习、复习中、已掌握、暂停复习。

## 7.2 复习调度模块

- 系统不强制孩子在软件中学新词；家长一旦标记“已学过”，系统即开始安排滚动复习。

- 默认复习节奏建议为：第 0 天、第 1 天、第 3 天、第 7 天、第 14 天、第
30 天。

- 如答错：当天稍后再次出现，次日优先复习；连续答错 2
次以上进入红色高频池。

- 每日任务生成优先级：到期红词 \> 到期黄词 \> 到期会写词 \> 历史错词回炉
\> 绿色词抽查。

- 支持设置每日任务上限；如到期单词过多，低优先级任务顺延到次日。

## 7.3 练习模块

- 认读词 V1.0 题型建议：看英文选中文、看中文选英文。

- 会写词 V1.0 题型建议：看中文拼英文、补全单词（缺字母）。

- 每题提交后立即给出正确 / 错误反馈；错误时显示正确答案。

- 当日做错单词需要在本次任务的后半段再次出现一次。

## 7.4 错词本模块

- 系统自动收集所有做错过的词。

- 每条错词应展示：英文、中文、错误次数、最近错误时间、当前掌握等级、词类型、最近一次错误类型。

- 支持按全部错词、认读错词、会写错词、近 7
天错词、红色高危词等条件筛选。

## 7.5 每周管理与统计模块

- 支持以“周”为单位管理单词，如第 1 周、第 2 周、第 3 周。

- 每周统计需展示：本周录入单词数、认读词数、会写词数、已掌握数量、未掌握数量、高危错词数量。

- 每周应支持查看：本周最容易错的 10
个词、本周会写最不稳的词、本周已掌握词、需带入下周继续复习的词。

# 8. 掌握分层与晋级规则

| **单词类型** | **掌握层级** | **建议晋级规则**       | **建议复习频率** |
|--------------|--------------|------------------------|------------------|
| 认读词       | 红色         | 连续答对 2 次转黄      | 1 天后复习       |
| 认读词       | 黄色         | 连续答对 3 次转绿      | 3 天后复习       |
| 认读词       | 绿色         | 答错降黄；连续错可降红 | 7-14 天抽查      |
| 会写词       | 红色         | 连续拼写正确 2 次转黄  | 1 天后复习       |
| 会写词       | 黄色         | 连续拼写正确 3 次转绿  | 2 天后复习       |
| 会写词       | 绿色         | 答错降黄；连续错可降红 | 5-7 天复习       |

# 9. 页面需求

**首页：**显示今日任务总数、认读任务数、会写任务数、错词回炉数、今日完成进度、连续打卡天数。

**单词录入页：**支持手动新增、批量导入、标记认读 / 会写、设置所属周次。

**单词列表页：**支持按周次、类型、掌握等级筛选查看。

**今日任务页：**孩子进入后可直接开始复习，由系统按优先级出题。

**错词本页：**展示所有错词和错误次数，可筛选高危词。

**统计页：**展示每日、每周、累计掌握情况。

**设置页：**支持设置任务量、复习节奏和是否开启绿色词抽查。

字段建议：
id，word_id，review_time，question_type，user_answer，correct_answer，is_correct，response_time，error_type

字段建议： id，week_no，batch_name，created_at

## daily_tasks

字段建议： id，task_date，word_id，task_type，priority，status

# 11. 交互与非功能要求

- 界面要简单，适合孩子操作；字体清晰，按钮较大，减少复杂路径。

- 进入系统后可直接开始“今日任务”，避免层层跳转。

- 页面响应要快，数据本地保存稳定；后续预留云同步能力。

- 优先支持手机和平板，也可先做网页 / PWA 版本以加快开发。

# 12. 技术建议

- 如需快速落地，优先建议开发网页 / PWA 版本，便于手机和平板直接打开。

- 前端可采用 Vue 或 React；本地存储可采用 SQLite 或 IndexedDB。

- 如后续需要打包正式 App，可考虑 Expo / React Native。

# 13. 验收标准

- 家长能够录入本周 100 个单词，并标记 30 个会写词、70 个认读词。

- 系统能够自动生成每日复习任务，并在第二周继续带入前一周未掌握词。

- 孩子能够完成认读练习与拼写练习，错词会自动进入回炉池。

- 系统能够根据答题结果动态调整掌握等级和下次复习时间。

- 每日任务量能够被控制，不会随着周数增加而无限膨胀。

- 家长能够查看错词、掌握情况和每周复盘结果。

# 14. 给开发人员的最终说明

本项目不是做一个完整英语学习平台，**而是做一个面向 KET
备考家庭、以旧词滚动复习和会写词强化为核心的小型单词管理系统。**
孩子的新词学习过程可以在线下完成，系统负责把“学过的词”纳入科学复习，解决越学越多、旧词遗忘、会写词难巩固的问题。
