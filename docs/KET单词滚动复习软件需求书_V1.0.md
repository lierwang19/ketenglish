# KET 单词滚动复习软件需求书

V1.0｜面向 KET 备考家庭的纸质任务单复习与错词回炉系统

| **项目定位** | 家长端旧词滚动复习机 + 纸质任务单管理工具                    |
| ------------ | ------------------------------------------------------------ |
| **核心对象** | KET 备考中的小学生家庭                                       |
| **使用方式** | 孩子保留自己的新词记忆方式，系统负责周录入、任务单生成、拍照批改与长期滚动复习 |
| **交付对象** | 可直接提供给前端 / 后端 / 产品 / 测试开发人员作为 V1.0 实施依据 |

一句话总结：本项目不是完整英语学习平台，而是一个围绕老师每周词表、通过每日纸质任务单持续安排旧词复习与错词回炉的小型家庭工具。

# 1. 项目背景

- 孩子正在备考 KET，在校老师每周会布置约 100 个单词任务，其中约 30 个要求会写，定义为“基础词汇”；约 70 个要求看到英文能说出中文，定义为“拓展词汇”。

- 孩子已经形成自己的新词学习方式，因此产品不替代新词学习过程，而是重点解决“背过就忘、旧词越积越多、会写词难巩固、家长难安排复习、纸上作答难沉淀数据”的问题。

- 家长更容易接受“系统生成表格 -> 打印 -> 孩子纸上作答 / 复习 -> 拍照上传 -> 更新掌握状态”的方式，因此 V1.0 的核心闭环应以纸质任务单为中心，而不是以孩子在系统内逐题作答为中心。

# 2. 项目目标

- 支持家长每周录入老师布置的新单词，并区分基础词汇与拓展词汇。

- 根据复习规则自动生成每日任务单，并支持导出为可打印 PDF。

- 让孩子通过纸面完成基础词默写与拓展词复习，家长用拍照上传结果的方式回收作答与批改信息。

- 支持基础词自动批改、拓展词人工校对结果入库，并根据批改结果更新单词掌握情况与下次复习时间。

- 对错词进行自动回炉，对已稳定掌握词降低频率，避免第二周、第三周后任务量失控。

- 提供家长可视化统计，便于查看周录入量、掌握情况、错词情况和每周复盘结果。

# 3. 产品核心原则

- 目标上对齐老师要求，执行上采用“每周录入 + 每日少量推进 + 长期滚动复习”的方式。

- 尊重孩子现有的新词学习方法，系统主要接管旧词复习、纸质任务单生成、拍照批改和错词回炉。

- 基础词汇与拓展词汇必须分开管理，采用不同复习频率、不同任务单展示方式和不同批改方式。

- 任务量必须可控。系统不追求“所有到期词当天全部出现”，而是追求“在固定预算内优先复习最需要复习的词”。

- 产品界面与交互默认以家长使用为主，孩子在 V1.0 中不是主要操作端。

# 4. 用户角色与使用边界

## 4.1 家长端

- 录入本周单词，支持手动录入、表格导入、截图导入入口。

- 标记单词类型：基础词汇 / 拓展词汇。

- 配置每日任务量、提醒时间和复习参数。

- 生成当日任务单，下载 / 打印 PDF。

- 孩子纸上完成后，家长上传照片，确认基础词自动批改结果，并对拓展词进行纸面勾叉批改后入库。

- 查看单词本、任务单状态、错词情况、每周统计与掌握分层。

## 4.2 孩子端

- V1.0 不要求孩子在系统中直接完成题目。

- 孩子主要在纸上完成基础词书写与拓展词复习。

- 积分兑换、激励玩法可作为后续扩展方向，不进入本次核心闭环。

## 4.3 V1.0 边界

- V1.0 不要求在系统中完成完整新词学习。

- V1.0 不做 AI 讲解、口语评分、复杂游戏机制、多人社交排名。

- V1.0 不把“直接发送 PDF 到个人微信”定义为硬性能力，普通微信送达受外部生态限制；企业微信发送作为 P1 增强项。

- V1.0 的重点是把“已经学过的词”纳入长期可持续的复习系统。

# 5. 核心业务流程

- 家长按周录入老师布置的新单词，支持手动输入、表格导入，并预留截图自动导入入口。每个单词分为基础词汇或拓展词汇。

- 单词属性遵循以下规则：
  - 基础词汇一旦建立，不自动降级为拓展词汇。
  - 拓展词汇如果后续再次录入为基础词汇，则以基础词汇为准。

- 家长点击“生成今日任务单”，系统根据复习规则生成当日任务单：
  - 基础词汇按家长设置的数量生成，平均分成左右两列。左列为“中文写英文”，右列为“英文写中文”。
  - 拓展词汇按家长设置的数量生成，左右各三列表格，每条包含“序号、单词、中文释义、批改”。

- 家长下载 / 打印任务单，孩子在纸上完成基础词填写，拓展词部分由系统完整展示词义供复习和家长批改。

- 家长拍照上传任务单：
  - 基础词部分由系统识别孩子填写的答案，并进行自动判定。
  - 拓展词部分由家长先在纸面标记勾 / 叉，系统识别家长批改标记。

- 系统将批改结果计入掌握状态、累计错误次数和下次复习时间，并将错词加入高优先级回炉池。

- 已稳定掌握的单词进入低频抽查池，不再长期高频出现。

# 7. 详细功能需求

## 7.1 单词录入与周管理

- 支持手动新增单词，核心字段包括：英文、中文、单词类型、所属周次、录入时间、备注。

- 支持表格导入，最少字段为：英文、中文、单词类型、所属周次。

- 保留截图导入入口，V1.0 可先以占位与后续接入说明呈现，但产品口径仍定义为 P0 目标。

- 支持编辑单词内容、所属周次、备注以及单词类型。

- 每个单词至少需记录：
  - 所属周次
  - 单词类型（基础词汇 / 拓展词汇）
  - 录入时间
  - 当前掌握层级
  - 累计错误次数

## 7.2 单词本模块

- 单词本必须区分“基础词汇”“拓展词汇”。

- 默认支持按录入周数筛选，例如“第 1 周”“第 2 周”。

- 默认采用单周独立查看，不要求 V1.0 提供跨周对比视图。

- 筛选某一周后，页面按两部分表格展示：
  - 第一部分：基础词汇
  - 第二部分：拓展词汇

- 每个单词至少展示：
  - 英文
  - 中文
  - 录入时间
  - 累计错误次数
  - 当前掌握层级

## 7.3 每日任务生成

- 系统每日生成一张任务单。

- 基础词和拓展词的每日数量由家长在设置中调整，不把固定数值写死为不可变业务规则。

- 推荐默认值可参考：
  - 基础词汇：6 个 / 日
  - 拓展词汇：15 个 / 日

- 当日任务量采用“固定预算 + 智能补位”机制：
  - 先抽高优先级候选词
  - 不足再补位
  - 超出则顺延到后续日期

- 错词回炉占用当天配额，不额外扩容，以确保任务总量可控。

## 7.4 每日任务单 PDF 结构

- 输出格式为可下载 PDF，方便家长直接打印。

- 整体版式参考老师纸面词表风格，采用清晰、规整、适合手写和纸面批改的表格布局。

- 页面分成两部分：
  - 第一部分：基础词汇任务区
  - 第二部分：拓展词汇复习区

- 当日词量过多时自动续页，不通过极端缩小字体强行塞进单页。

## 7.5 基础词汇任务区版式

- 基础词汇区按家长设置的数量生成，并平均分成左右两列。

- 左列采用“中文写英文”模式，表格列为：`序号 | 中文释义 | 单词`。

- 右列采用“英文写中文”模式，表格列为：`序号 | 单词 | 中文释义`。

- 左右两列表格样式保持一致，只是出题方向不同。

- 该部分必须保留足够书写空间，方便孩子纸面填写。

## 7.6 拓展词汇任务区版式

- 拓展词汇区也采用左右分栏布局。

- 每侧表格结构为：`序号 | 单词 | 中文释义 | 批改`。

- 系统直接展示英文和中文，不要求孩子填写。

- “批改”列用于家长纸面勾叉或标记掌握情况，供后续拍照识别。

## 7.7 上传与批改

- 基础词部分：系统识别孩子书写结果并判定正误。

- 拓展词部分：系统识别家长在“批改”列中的勾 / 叉 / 标记结果。

- 批改结果必须支持人工复核和手工修正，避免 OCR 误判直接污染掌握数据。

- 任务单状态建议至少包括：
  - 待生成
  - 已生成
  - 已打印
  - 已上传
  - 待复核
  - 已完成

## 7.8 复习曲线与抽词机制

- 系统不采用“所有到期词当天必须全部出题”的策略，而采用“固定预算 + 候选池优先级评分 + 加权抽样”的调度机制。

- 每个单词每日生成一个抽取优先级，优先级至少由以下因素共同决定：
  - 是否到期
  - 逾期天数
  - 单词类型
  - 累计错误次数
  - 最近连续错误次数
  - 最近连续正确次数
  - 掌握层级
  - 最近一次复习时间

- 抽取顺序建议为：
  - 到期基础词
  - 高优先级错词回炉
  - 到期拓展词
  - 低频抽查词

- 池内抽词规则建议为：
  - 先按优先级分数排序
  - 再在高分区间内做小范围加权抽样
  - 未被抽中的到期词获得逾期补偿，提高次日被抽中的概率

- 默认曲线建议写为“推荐参数，可在设置中调整”，例如：
  - 基础词汇：D1 -> D3 -> D6 -> D10 -> D17 -> D30
  - 拓展词汇：D1 -> D4 -> D8 -> D15 -> D30

- 若答错：
  - 次日优先回炉
  - 连续错 2 次及以上进入高优先级池

- 已稳定掌握词进入低频抽查池，不再持续高频出现。

## 7.9 定时提醒与微信发送

- V1.0 P0：
  - 支持每日定时生成任务单
  - 支持站内提醒家长下载 / 打印任务单

- V1.0 不把“直接发送 PDF 到个人微信”写成硬承诺。

- P1：
  - 支持接入企业微信，定时向家长发送任务通知或 PDF 文件

## 7.10 错词回炉与掌握更新

- 批改完成后，系统统一更新：
  - 掌握层级
  - 累计错误次数
  - 最近一次结果
  - 下次复习时间

- 错词自动进入高优先级回炉池。

- 已稳定掌握词进入低频复习池。

- 系统不再使用“当次系统内答错，立刻在本轮后半段再出现”的旧逻辑。

## 7.11 统计与周复盘

- 面向家长展示以下核心信息：
  - 本周录入单词数
  - 本周基础词汇数
  - 本周拓展词汇数
  - 已掌握词数量
  - 学习中词数量
  - 红词数量
  - 累计错误次数较高的单词

- 每周应支持查看：
  - 本周最易错词
  - 基础词最不稳定词
  - 待带入下周继续复习的词

# 8. 掌握分层与晋级规则

| **单词类型** | **掌握层级** | **建议晋级规则**               | **建议复习频率** |
| ------------ | ------------ | ------------------------------ | ---------------- |
| 拓展词汇     | 红色         | 连续通过 2 次任务单转黄        | 次日 / 3 日内    |
| 拓展词汇     | 黄色         | 连续通过 3 次任务单转绿        | 4-8 日           |
| 拓展词汇     | 绿色         | 答错降黄；连续错可降红         | 15-30 日抽查     |
| 基础词汇     | 红色         | 连续书写正确 2 次转黄          | 次日 / 3 日内    |
| 基础词汇     | 黄色         | 连续书写正确 3 次转绿          | 6-10 日          |
| 基础词汇     | 绿色         | 答错降黄；连续错可降红         | 17-30 日抽查     |

说明：

- 上述规则是推荐默认参数，允许在设置中调整。

- 对外展示可以保持“红 / 黄 / 绿”，内部调度可使用更细致的分数或稳定度参数。

# 9. 页面需求

**首页：**

- 展示今日任务单状态：待生成、已生成、待打印、待上传、待复核、已完成
- 展示本周基础词汇数、拓展词汇数、高风险词数量
- 提供快捷入口：录入单词、查看任务单、上传与批改、查看单词本

**单词录入页：**

- 支持周次管理
- 支持批量录入基础词汇 / 拓展词汇
- 预留截图导入入口

**单词本页：**

- 支持按录入周次筛选
- 筛选后按“基础词汇”“拓展词汇”分区表格展示

**今日任务单页：**

- 展示当日任务单预览
- 支持导出 / 打印 PDF
- 展示任务单状态与历史记录

**上传与批改页：**

- 支持上传任务单照片
- 展示识别结果、人工修正入口和任务状态更新

**统计页：**

- 展示掌握分层、周录入结构、最近复习记录、本地存储情况

**设置页：**

- 支持设置基础词 / 拓展词每日数量
- 支持设置默认复习曲线、提醒时间和是否开启低频抽查
- 预留企业微信发送开关

# 11. 交互与非功能要求

- 界面要简单，主要面向家长操作，减少无效跳转。

- 优先支持手机和平板，也可先做网页 / PWA 版本以加快落地。

- 任务单导出与打印必须清晰稳定，适合 A4 纸打印。

- 图片上传与识别流程要考虑家长真实拍照环境，允许轻微倾斜、阴影和纸张折痕。

- 数据默认本地保存稳定，后续可预留云同步能力。

- 定时提醒与任务状态更新应保持幂等，避免重复生成不一致的任务单。

# 12. 技术建议

- 前端优先采用网页 / PWA 版本，方便手机和平板直接打开。

- 本地数据可采用 IndexedDB；如后续有服务端，可升级为云端同步架构。

- 任务单 PDF 可优先采用浏览器打印 / 导出能力，后续再接入更稳定的服务端 PDF 生成。

- OCR / 图像识别链路建议按“基础词自动识别 + 拓展词批改标记识别”分阶段接入。

- 企业微信推送能力作为 P1，可在任务单状态与定时任务能力稳定后接入。

# 13. 验收标准

- 家长能够录入本周约 100 个单词，并区分基础词汇与拓展词汇。

- 单词本能够按录入周数筛选，并分基础词汇 / 拓展词汇两部分表格展示。

- 单词本每个单词能够展示录入时间和累计错误次数。

- 系统能够自动生成每日任务单，并支持导出 / 打印 PDF。

- 基础词汇任务区能够左右分栏展示“中文写英文”“英文写中文”。

- 拓展词汇任务区能够展示“序号、单词、中文释义、批改”。

- 家长能够上传任务单照片，并记录上传、待复核、已完成等任务状态。

- 系统能够根据批改结果动态调整掌握等级和下次复习时间。

- 每日任务量能够被控制，不会随着周数增加而无限膨胀。

- 家长能够查看本周录入量、掌握情况、错词和每周复盘结果。

# 14. 给开发人员的最终说明

本项目不是做完整英语学习平台，**而是做一个面向 KET 备考家庭、以纸质任务单为中心的旧词滚动复习与基础词强化系统。**

孩子的新词学习过程在线下完成，系统负责把“已经学过的词”纳入长期可持续的复习闭环：周录入、任务单生成、纸面作答、拍照批改、状态更新与错词回炉。  
开发实现时，应始终优先保证以下三件事：

- 任务单结构清晰、可打印、易于家长执行
- 任务量长期可控，不因词库累积而失控
- 页面结构与数据口径始终服务于“家长端纸质闭环”，而不是回退到“系统内逐题练习”旧模型
